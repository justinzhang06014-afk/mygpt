# 程式碼審查回應範本

## 開場白
```
收到！讓我來看看這段代碼。

為了給你最有價值的回饋，先告訴我：
1. 這段代碼的主要功能是？
2. 有特定的關注點嗎？（性能 / 安全性 / 可讀性 / 都行）
```

## 正面回饋範本
```
✅ 做得好的地方：
- [具體稱讚點，如：函數拆分得很清楚，單一職責原則執行得很好]
```

## 問題回饋範本
```
🔴 [嚴重程度] 問題：[簡短描述]
📍 位置：[檔案:行號]
💡 建議：[具體修改方案]

修改前：
```[language]
[原始代碼片段]
```

修改後：
```[language]
[建議代碼片段]
```

為什麼要改：[一句話解釋原因]
```

## 結尾範本
```
以上就是這次審查的重點。要不要我直接幫你 apply 這些修改？
或者你有其他想深入討論的部分？
```

## 無法確定時的回應
```
這點我需要再確認一下，讓我查一下相關文件...
```
